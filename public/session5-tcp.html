Lập trình Sockets TCP

Mục tiêu bài học

  • Hiểu TCP: kết nối, tin cậy, có thứ tự, hướng dòng (byte-stream), không biên thông điệp.
  • Nắm đối tượng ServerSocket, Socket và vòng đời kết nối: bind → listen → accept → read/write → close.
  • Làm việc với luồng text và nhị phân; lựa chọn charset, cách đóng gói thông điệp (framing).
  • Thiết lập timeout, xử lý ngoại lệ thường gặp, và quản lý tài nguyên đúng cách.
  • Thực hành 1 mẫu: server/client tính toán đơn giản qua TCP.

1. Lý thuyết bổ sung

  • TCP là byte‑stream: không có biên thông điệp; ứng dụng phải tự xác định ranh giới (ví dụ kết thúc dòng, tiền tố độ dài...).
  • Địa chỉ socket: cặp <IP:port> ở cả phía local và remote. Port 0 cho phép hệ thống cấp port tạm thời (ephemeral).
  • Bắt tay 3 bước (SYN, SYN‑ACK, ACK) khi kết nối; đóng: FIN/ACK, có thể half‑close (đóng 1 chiều) với shutdownOutput()/shutdownInput().
  • Blocking I/O: accept(), read()/readLine() chặn cho tới khi có sự kiện; dùng setSoTimeout(ms) để đặt timeout đọc; connect(addr, timeout) cho kết nối.
  • Buffering & flush: với giao thức dạng dòng, dùng PrintWriter bật autoFlush để đẩy dữ liệu khi gọi println.
  • Charset & EOL: nên chỉ định charset (UTF‑8). Quy ước kết thúc dòng phổ biến là \n hoặc \r\n tùy giao thức.

Ghi nhớ: TCP đảm bảo thứ tự và độ tin cậy ở mức byte. Việc phân mảnh thông điệp thuộc về tầng ứng dụng — hãy tự định nghĩa biên thông điệp rõ ràng.

2. Ví dụ duy nhất — Máy chủ/khách thực hiện phép tính

Giao thức đơn giản: client gửi 1 dòng có dạng so1@pheptoan@so2 (ví dụ 12@+@5), server đọc và trả về kết quả trên 1 dòng.

package com.mycompany.demo1;

import java.io.IOException;
import java.io.PrintWriter;
import java.net.ServerSocket;
import java.net.Socket;
import java.util.Scanner;

/**
 * @author PC
 */
public class TCPServer {
  public static void main(String[] args) throws IOException {
    int PORT = 1234;
    ServerSocket server = new ServerSocket(PORT);
    System.out.println("Server listening");
    try {
      Socket socket;
      while ((socket = server.accept()) != null) {
        Scanner in = new Scanner(socket.getInputStream());
        PrintWriter out = new PrintWriter(socket.getOutputStream(), true);

        String chuoi = in.nextLine().trim();
        in = new Scanner(chuoi);
        in.useDelimiter("@");

        int so1 = in.nextInt();
        String pheptoan = in.next();
        int so2 = in.nextInt();

        switch (pheptoan) {
          case "+": out.println(so1 + so2); break;
          case "-": out.println(so1 - so2); break;
          case "*": out.println(so1 * so2); break;
          case "/": out.println((float) so1 / so2); break;
          default:   out.println("Phep toan khong hop le");
        }
      }
    } catch (Exception e) {
      e.printStackTrace();
    }
  }
}

package com.mycompany.demo1;

import java.io.IOException;
import java.io.PrintWriter;
import java.net.Socket;
import java.util.Scanner;

/**
 * @author PC
 */
public class TCPClient {
  private static Socket socket = null;
  private static PrintWriter out = null;

  public static void main(String[] args) {
    Socket socket = null;
    PrintWriter out = null;
    Scanner in = null;

    int so1, so2; String pheptoan, chuoi, ketqua;

    in = new Scanner(System.in);
    out = new PrintWriter(System.out, true);

    out.println("Nhap so thu 1:");
    so1 = in.nextInt();
    out.println("Nhap so thu 2:");
    so2 = in.nextInt();
    out.println("Nhap phep toan:");
    pheptoan = in.next();

    chuoi = so1 + "@" + pheptoan + "@" + so2;

    try {
      socket = new Socket("127.0.0.1", 1234);
      out = new PrintWriter(socket.getOutputStream(), true);
      in = new Scanner(socket.getInputStream());

      out.println(chuoi);
      ketqua = in.nextLine().trim();
      System.out.println("Ket qua:" + ketqua);
      socket.close();
    } catch (IOException e) {
      if (socket != null) {
        try { socket.close(); } catch (IOException ex) {}
      }
    }
  }
}

Giải thích chi tiết

  • Client ghép chuỗi theo định dạng so1@pheptoan@so2 và gửi 1 dòng qua TCP; server đọc 1 dòng, tách bằng dấu @ và thực hiện phép tính.
  • Sử dụng PrintWriter(..., true) để auto‑flush khi gọi println, đảm bảo dữ liệu được đẩy đi ngay.
  • Ví dụ này là đơn luồng, server phục vụ tuần tự từng kết nối. Để phục vụ nhiều client đồng thời, áp dụng kỹ thuật ở Buổi 7 (thread/thread pool).
  • Vì TCP là byte‑stream, việc chọn ranh giới thông điệp dựa vào ký tự xuống dòng; nếu gửi nhiều phép tính trong một kết nối, tiếp tục gửi từng dòng.

3. Lưu ý và thực hành tốt

  • Chuẩn hóa input: giới hạn toán tử hợp lệ (+, -, *, /), tránh lỗi định dạng gây InputMismatchException.
  • Xử lý ngoại lệ và đóng tài nguyên: cân nhắc dùng try-with-resources để tự động đóng socket/luồng.
  • Nếu triển khai thực tế: thêm timeout đọc/kết nối để tránh treo khi peer không phản hồi.

Tổng kết

Ví dụ máy chủ/khách tính toán minh họa rõ cách đóng gói thông điệp dạng dòng qua TCP và cách xử lý ở server. Từ đây bạn có thể mở rộng để hỗ trợ nhiều phép tính, nhiều client đồng thời, hoặc thiết kế giao thức rõ ràng hơn.
